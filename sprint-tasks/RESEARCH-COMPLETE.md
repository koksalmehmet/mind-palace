## ğŸ”‘ Key Concepts Explained Simply

### Signals
**What:** Values that notify when they change  
**Why:** No boilerplate subscriptions, automatic cleanup  
**Simple Example:**
```typescript
const count = signal(0);           // Create
const doubled = computed(() => count() * 2);  // Derive
count.set(5);                      // Update
console.log(doubled());            // Get: 10
```

### OnPush Change Detection
**What:** Only update component when inputs change  
**Why:** 70% fewer updates, better performance  
**Result:** With signals, automatic integration

### Lazy Loading Routes
**What:** Load route components only when user visits  
**Why:** Smaller initial download, faster startup  
**Example:**
```typescript
{ path: 'analysis', loadComponent: () => import('./analysis') }
```

### Vitest
**What:** Modern, fast test runner (Jest-compatible)  
**Why:** 5x faster, better DX, ESM-native  
**Switch From:** Karma (slower, less DX)

### TypeScript Strict
**What:** Compiler flags that enforce type safety  
**Why:** Prevent entire class of runtime bugs  
**Current:** 95% â†’ **Target:** 100%
